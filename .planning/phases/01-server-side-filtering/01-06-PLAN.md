<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL → RSC searchParams | nuqs-parsed filter params cross the server boundary; plantSearchParamsCache validates |
| RSC → Supabase | buildPlantsQuery translates typed params to parameterized SQL; no string concatenation except .or() for q param |
| PlantsGrid → Server Action | fetchMorePlants accepts PlantSearchParams (typed); creates new Supabase server client |
| User → create list modal | List creation requires authenticated user (verified via supabase.auth.getUser() inside handler) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | URL params → plantSearchParamsCache.parse() | mitigate | nuqs parsers reject non-string, apply defaults, prevent null injection; all values then flow through parameterized Supabase queries |
| T-06-02 | Tampering | params.q ilike string interpolation in .or() | mitigate | parseAsString (nuqs) normalizes input to a plain string; PostgREST encodes the value as a URL query param (not raw SQL); no injection path confirmed. SRCH-01 (v2) will replace with tsvector and eliminate this entirely |
| T-06-03 | Elevation of Privilege | handleCreateList creates list for authenticated user | mitigate | supabase.auth.getUser() called inside handler; list inserted with owner_id = user.id; Supabase RLS INSERT policy requires auth.uid() = owner_id — double enforcement |
| T-06-04 | Information Disclosure | PlantsGrid fetches user's lists (passed from RSC) | accept | Lists are user-scoped (RLS enforced server-side); RSC fetches only if user is authenticated; passed as prop |
| T-06-05 | Denial of Service | COUNT query runs on every RSC re-render | accept | COUNT with filters is fast at current database size; Supabase connection pooling handles concurrent requests; no rate limit needed at this scale |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
