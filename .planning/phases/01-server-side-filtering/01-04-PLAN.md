<!-- Supabase query operators (from RESEARCH.md Pattern 3): -->
.in('field', array)                               → OR match on scalar field
.overlaps('field', array)                          → OR match on array field (any element matches)
.contains('field', array)                          → AND match on array field (all elements present)
.lte('field', value) .gte('field', value)          → numeric range comparison
.or('common_name.ilike.%q%,latin_name.ilike.%q%') → text search across two columns
.range(from, to)                                   → pagination (0-indexed, inclusive)
.select('*', { count: 'exact', head: true })       → count only, no rows
.order('common_name')                              → alphabetical sort (applied before range)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | URL params → buildPlantsQuery | mitigate | PlantSearchParams typed as string arrays; nuqs parsers validated these before reaching actions.ts; Supabase JS uses parameterized queries — no SQL injection path via .in(), .contains(), .overlaps(), .lte(), .gte() |
| T-04-02 | Tampering | params.q used in .or() with string interpolation | mitigate | params.q is parseAsString (nuqs-validated string); .or() passes value as PostgREST filter expression; Supabase JS client encodes it as a URL query param, not SQL string concatenation; risk is low. Note: full-text search via tsvector (SRCH-01, v2) would eliminate this path entirely |
| T-04-03 | Information Disclosure | fetchMorePlants returns Plant[] with all columns | accept | plants table has RLS SELECT USING(true) — fully public; no PII in plant records |
| T-04-04 | Denial of Service | Rapid "load more" clicks fire multiple Server Action calls | accept | Next.js 14 Server Actions are serialized per session; nuqs throttles URL updates 50ms by default; acceptable at current traffic scale |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
