      const [isPending, startTransition] = useTransition()

      handleToggle(key: keyof typeof filters, value: string):
        const current = filters[key] as string[]
        const next = current.includes(value) ? current.filter(v =&gt; v !== value) : [...current, value]
        setFilters({ [key]: next })

      handleClearAll():
        setFilters({ sun: [], water: [], types: [], months: [], dormancy: [], growthRate: [], layers: [], permUses: [], zones: [], state: '', q: '' })

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User interaction → nuqs URL write | Filter toggles write URL via useQueryStates; URL becomes user-controlled input to RSC |
| FilterControls → RSC | shallow:false causes RSC re-render with new searchParams |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | URL params set by FilterControls | mitigate | Values are constrained to known option arrays (SUN_OPTIONS, etc.); plantSearchParamsCache.parse() in RSC validates and sanitizes before Supabase query |
| T-05-02 | Denial of Service | Rapid filter toggling fires rapid RSC re-renders | accept | nuqs throttles URL updates (50ms default); React useTransition serializes RSC requests; acceptable at scale |
| T-05-03 | Tampering | ActiveFilterChips setFilters({sun: []}) — could be called by XSS | accept | nuqs setFilters only writes URL params; no server side-effect; data is public plant catalog |
| T-05-04 | Tampering | Mobile drawer "Show N plants" button | accept | Closes drawer only — no data mutation |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
