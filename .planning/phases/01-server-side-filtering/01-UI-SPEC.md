## Interaction Contracts

### Visual Hierarchy

The plant card grid is the primary focal point; the filter sidebar is secondary. On desktop, the eye enters top-left at the page heading, moves through active filter chips and result count, then lands on the card grid. On mobile, the sticky "Filters" button anchors the entry point.

### Filter Behavior

- Filter change triggers immediate URL update via nuqs `useQueryState` â€” no debounce, no "Apply" button.
- Any filter change resets pagination to page 1 and replaces the loaded result set entirely (D-08).
- Desktop sidebar: always visible, sticky, `max-h-[calc(100vh-7rem)] overflow-y-auto`.
- Mobile: collapsible drawer below the "Filters" toggle button. Drawer footer shows "Show N plants" (forest bg) and "Clear all" (warm-stone text, terracotta hover).
- Desktop and mobile both render the same `FilterControls` component â€” single source of truth (D-10).

### Active Filter Chips

Chips appear in two locations simultaneously:
1. **Inside the filter sidebar** (existing pattern) â€” at the top, above filter sections, bordered below with `border-warm-stone/20`.
2. **Above the results grid** (new in this phase) â€” between the search bar and the result count line. Visible on both desktop and mobile.

Chip anatomy:
- Background: `bg-terracotta`
- Text: `text-white text-[11px] capitalize`
- Shape: `rounded-full`
- Padding: `px-3 py-1`
- Remove control: `Ã—` suffix, `text-white/70`, becomes `text-white` on hover
- Hover state: `hover:bg-terracotta/80`
- "Clear all" link: `text-[11px] text-warm-stone hover:text-terracotta underline underline-offset-2`

Chip label format by filter type:
- Sun: `full sun` / `part shade` / `full shade` (capitalize)
- Water: `low water` / `moderate water` / `high water`
- Type: `shrub` / `tree` / etc. (capitalize)
- USDA Zone: `Zone 9a` / `Zone 9b`
- Native State: `Native to California` (full state name, not code)
- Permaculture Use: `nitrogen fixer` (capitalize)
- Forest Layer: `canopy` / `shrub` / etc. (capitalize)
- Bloom Month: `January` / `February` / etc.

### Skeleton Loading State

Shown while the server-side query is pending (Suspense fallback during filter changes and initial load).

Skeleton card anatomy:
- Count: 24 cards (matches page size D-06)
- Background: `bg-stone-white` (not cream â€” slightly dimmer than card surface)
- Shape: `rounded-2xl` (16px) â€” matches real card
- Border: `border border-warm-stone/20`
- Image placeholder: `h-52 bg-stone-white/80 rounded-t-2xl` â€” no animation content
- Title bar: `h-5 w-3/4 bg-warm-stone/20 rounded mt-5 mx-5`
- Subtitle bar: `h-3 w-1/2 bg-warm-stone/15 rounded mt-2 mx-5`
- Badge row: two `h-6 w-16 bg-warm-stone/15 rounded-full` blocks, `mt-3 mx-5 flex gap-2`
- Animation: `animate-pulse` (Tailwind built-in) â€” gentle fade only, no shimmer

Grid layout during skeleton: identical to loaded state â€” `grid sm:grid-cols-2 xl:grid-cols-3 gap-6`.

### Result Count Line

Position: between active filter chips row and the plant grid.

Copy format: `Showing {loaded} of {total} plant{s}` where:
- `{loaded}` = number of plant cards currently rendered (cumulative after "Load more")
- `{total}` = COUNT query result with same filters applied
- Pluralization: `plant` when total is 1, `plants` otherwise

Typography: `text-sm font-normal text-warm-umber mb-5`

### Load More Button

Shown only when `loaded < total`. Hidden when all results are rendered.

Active state (more pages available):
- Label: `Load more plants`
- Style: `text-sm font-medium border border-warm-stone/40 text-warm-umber py-2 px-6 rounded-lg hover:bg-stone-white transition-colors duration-150`
- Position: centered below the grid, `mt-10`
- Loading state: label changes to `Loadingâ€¦`, button `disabled:opacity-50 disabled:cursor-not-allowed`

Exhausted state (all results loaded, total > 24):
- Label: `All {total} plants loaded`
- Style: `text-sm text-warm-stone text-center mt-10` â€” no button, plain text only

When total <= 24: no Load More button rendered at all.

### Empty State

Shown when server query returns 0 results with active filters.

Anatomy:
1. Heading: `No plants match your filters.` â€” Inter 16px, `text-warm-umber`, centered
2. Body: `Try removing a filter to broaden your search.` â€” Inter 14px, `text-warm-stone`
3. Active filter chips: rendered in full below the body text (same chip component as above-grid chips) â€” user can selectively remove filters without clicking sidebar
4. "Clear all filters" button: `text-sm font-medium text-forest hover:text-forest-dark underline underline-offset-2`

Container: `py-20 text-center flex flex-col items-center gap-4`

### Error State

Shown if the Supabase query fails (network error, timeout).

Copy:
- Heading: `Couldn't load plants`
- Body: `There was a problem fetching the plant catalog. Check your connection and try again.`
- Action: `Try again` button â€” same style as Load More (secondary outlined)

Typography matches empty state. No inline stack traces shown to user.
