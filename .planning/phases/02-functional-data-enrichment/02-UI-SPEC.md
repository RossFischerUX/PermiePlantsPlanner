# Phase 2 — UI Design Contract

## Spacing Scale

Declared values (must be multiples of 4). These are the exact Tailwind utilities already in use on the detail page; the new sections MUST reuse them, not introduce new spacing.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | n/a this phase |
| sm | 8px | Pill flex gap (`gap-2`); pill vertical padding (`py-1.5` ≈ 6px — existing, retained for consistency with current `permaculture_uses` chips) |
| md | 12px | InfoCell grid gap (`gap-3`); pill horizontal padding (`px-3`) |
| md | 16px | InfoCell internal padding (`p-4`); section header bottom margin (`mb-4`) |
| lg | 24px | n/a this phase |
| xl | 32px | Inter-section vertical rhythm (`mb-8` between detail sections) |
| 2xl | 48px | n/a this phase |
| 3xl | 64px | n/a this phase |

**Section spacing contract:** Each new section uses `className="mb-8"` (32px) exactly like existing `Plant Overview` / `Landscaping` sections. Within a section: `mb-4` (16px) below the `h2` eyebrow; `gap-3` (12px) for InfoCell grids; `gap-2` (8px) for pill rows. When a section mixes scalar InfoCell rows and pill rows (Forest Layer & Succession, Establishment & Care, Harvest), the InfoCell grid block carries `mb-4` (16px) before the pill block — mirroring the existing Permaculture section's `mb-4` between its InfoCell grid and `permaculture_uses` pills.

Exceptions: pill `py-1.5` (6px) is a pre-existing non-grid value retained verbatim from current `permaculture_uses` chip markup for visual consistency — do not "correct" it to 8px.

## Copywriting Contract

This is a read-only display phase. There is no CTA, no form, no error path, no destructive action. The copy contract covers section titles, InfoCell labels, and conditional empty-state (hide) behavior.

### Section titles (rendered as `<h2>` eyebrows, in this exact order)

Detail page order: Plant Overview → Landscaping → **Functional Roles** → **Forest Layer & Succession** → **Establishment & Care** → **Harvest** → Notable Cultivars. The standalone "Permaculture" section is removed; its content is redistributed (CONTEXT.md D-05, Specifics).

| Section | Eyebrow copy (Title Case, rendered `uppercase` via CSS) | Source fields |
|---------|--------------------------------------------------------|---------------|
| 1 | `Functional Roles` | `permaculture_uses` (normalized 16-tag controlled vocab) |
| 2 | `Forest Layer & Succession` | `forest_garden_layer` (InfoCell), `succession_role` (pills) |
| 3 | `Establishment & Care` | `establishment_difficulty`, `maintenance_level`, `years_to_bearing` (InfoCells), `propagation_methods` (pills) |
| 4 | `Harvest` | `edible_parts` (pills), `harvest_months` (pills) |

### InfoCell labels (scalar fields — `<p>` label copy, Title Case)

| Field | InfoCell label copy |
|-------|---------------------|
| `forest_garden_layer` | `Forest Garden Layer` |
| `establishment_difficulty` | `Establishment` |
| `maintenance_level` | `Maintenance` |
| `years_to_bearing` | `Years to Bearing` |

Display label constants for the above (and any pill-group sub-labels) MUST be added to `lib/plant-labels.ts` (CONTEXT.md Claude's Discretion + Phase 1 D-09), not inlined in the page. InfoCell values inherit existing `capitalize` styling; enum values (`easy`/`moderate`/`challenging`, `low`/`moderate`/`high`, layer enum) display as-is, capitalized. `years_to_bearing` value renders as the integer plus the word `years` (e.g., `3 years`).

### Conditional / empty-state contract (CONTEXT.md D-07, Specifics)

No empty-state copy is shown. Missing data is **hidden**, never labeled "N/A" or "Unknown", reusing the existing `&&` guard pattern from the current Permaculture section.

| Condition | Behavior |
|-----------|----------|
| `permaculture_uses` empty/null | Hide entire **Functional Roles** section |
| `forest_garden_layer` null AND `succession_role` empty/null | Hide entire **Forest Layer & Succession** section |
| `forest_garden_layer` null (but succession present) | Render section; omit only the Forest Garden Layer InfoCell |
| `succession_role` empty/null (but layer present) | Render section; omit only the succession pill row |
| all of `establishment_difficulty`, `maintenance_level`, `years_to_bearing`, `propagation_methods` null/empty | Hide entire **Establishment & Care** section |
| `years_to_bearing` null | Omit only the Years to Bearing InfoCell row (non-food plants — never show an empty/zero value) |
| any individual scalar null | Omit only that InfoCell (filter-Boolean pattern, same as existing `overviewCells`) |
| `edible_parts` empty/null AND `harvest_months` empty/null | Hide entire **Harvest** section (plant is non-edible) |
| `edible_parts` empty (but `harvest_months` present) | Render section; omit only the edible-parts pill row |
| `harvest_months` empty (but `edible_parts` present) | Render section; omit only the harvest-months pill row |

### Pill ordering contract

- `harvest_months` pills MUST render in calendar order (Jan→Dec), reusing the existing `MONTH_OPTIONS` array in `lib/plant-labels.ts` for sort/ordering — identical to how `bloom_months` is treated (CONTEXT.md D-15). Do not render in raw stored/array order.
- All other tag arrays (`permaculture_uses`, `succession_role`, `propagation_methods`, `edible_parts`) render in stored array order (matches current `permaculture_uses` behavior — no sort applied).

| Element | Copy |
|---------|------|
| Primary CTA | not applicable — no CTA in this read-only phase |
| Empty state heading | not applicable — sections hide entirely when data absent (no empty-state UI) |
| Empty state body | not applicable |
| Error state | not applicable — RSC fetch already handled by existing `notFound()` guard; no new error surface |
| Destructive confirmation | not applicable — no destructive actions in this phase |
