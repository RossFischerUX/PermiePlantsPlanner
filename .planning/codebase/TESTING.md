# Testing Patterns
_Generated by gsd-codebase-mapper · focus: quality_

## Summary

The project uses Playwright E2E tests exclusively — no unit tests, no component tests, no Jest or Vitest. All tests run against the production Vercel URL (`https://permacultureplantpicker.com`). Tests are organized into four spec files covering auth, plants, lists, and presentation pages, with a global setup/teardown cycle that seeds and cleans test data in the production Supabase database.

---

## Test Framework

**Runner:** Playwright (version from `package.json`)
**Config:** `playwright.config.ts` at project root
**Assertion library:** Playwright built-in `expect`
**Target URL:** `https://permacultureplantpicker.com` (production — not localhost)

**Run commands:**
```bash
npx playwright test                    # Run all tests
npx playwright test tests/plants.spec.ts  # Run a single spec
npx playwright test --reporter=list    # Default reporter (already configured)
```

No watch mode or coverage commands — Playwright E2E only.

## Test File Organization

**Location:** `tests/` at project root (not co-located with source)

**Files:**
```
tests/
├── global-setup.ts         # Runs before all tests: signs in, seeds test data
├── global-teardown.ts      # Runs after all tests: deletes [TEST]* lists
├── auth.spec.ts            # Login, signup, signout, auth guard tests
├── plants.spec.ts          # Plant browser filters, search, detail page
├── lists.spec.ts           # My Lists CRUD, list editor actions
├── presentation.spec.ts    # Public /presents/ and /reports pages
├── .auth-state.json        # Cached browser storage state (gitignored)
├── .test-share-id.json     # Share ID + list ID from global setup (runtime)
└── .test-plant-ids.json    # Invasive + cultivars plant IDs (runtime)
```

**Naming convention:** `[feature].spec.ts`

## Playwright Projects (Auth Suites)

Two projects defined in `playwright.config.ts`:

| Project | Storage State | Spec Files |
|---------|--------------|------------|
| `logged-in` | `tests/.auth-state.json` | `lists.spec.ts`, `plants.spec.ts` |
| `logged-out` | Empty (no cookies) | `auth.spec.ts`, `presentation.spec.ts` |

`logged-out` depends on `logged-in` (runs after it so teardown ordering is correct).

Individual tests can override the project's storage state using `test.use({ storageState: ... })` — `plants.spec.ts` uses this to mix logged-in and logged-out tests in a single file:
```typescript
test.describe('Plant browser — public', () => {
  test.use({ storageState: { cookies: [], origins: [] } })
  // ...
})
test.describe('Plant browser — logged in', () => {
  test.use({ storageState: 'tests/.auth-state.json' })
  // ...
})
```

## Global Setup Pattern

`tests/global-setup.ts` runs once before all tests:

1. Signs in via the UI (Chromium) and saves browser storage state to `tests/.auth-state.json`
2. Creates a `[TEST] Presentation List` with 5 plants via the Supabase JS client
3. Writes `tests/.test-share-id.json` with `{ shareId, listId }` for use in presentation tests
4. Writes `tests/.test-plant-ids.json` with `{ invasivePlantId, cultivarsPlantId }` for conditional detail page tests

`tests/global-teardown.ts` deletes all lists matching `[TEST]%` owned by the test user.

**Test data convention:** Test-created records are always prefixed with `[TEST]` so teardown can reliably clean them:
```typescript
await supabase.from('plant_lists').delete().eq('owner_id', user.id).like('title', '[TEST]%')
```

**Auth credentials:** Loaded from `.env.test.local` via `dotenv` — `TEST_USER_EMAIL` and `TEST_USER_PASSWORD`. This file is not committed.

## Test Structure

**Suite organization with `test.describe`:**
```typescript
test.describe('Plant browser — public', () => {
  test.use({ storageState: { cookies: [], origins: [] } })

  test('loads plants with count badge', async ({ page }) => {
    await page.goto('/plants')
    await page.waitForSelector('p:has-text("Showing")', { timeout: 20000 })
    const count = await getFilteredCount(page)
    expect(count).toBeGreaterThan(0)
  })
})
```

**Fixture helpers:** Shared state loaded from JSON files at test runtime:
```typescript
function getTestIds(): { invasivePlantId: string | null; cultivarsPlantId: string | null } {
  try {
    return JSON.parse(fs.readFileSync(path.join('tests', '.test-plant-ids.json'), 'utf-8'))
  } catch {
    return { invasivePlantId: null, cultivarsPlantId: null }
  }
}
```

**Page helper functions:** Utility functions defined at the top of spec files, not in a shared module:
```typescript
async function getFilteredCount(page: Page): Promise<number> {
  const text = await page.locator('p', { hasText: 'Showing' }).first().textContent() ?? ''
  const match = text.match(/of (\d+) plant/)
  return match ? parseInt(match[1]) : 0
}
```

## Playwright Locator Patterns

**Filter sidebar interaction (must expand sections first):**
```typescript
// Expand a collapsed FilterSection before checking its children
await page.locator('aside button').filter({ hasText: 'Sun' }).click()
await page.locator('aside label').filter({ hasText: 'full sun' }).locator('input[type="checkbox"]').check()
```

**Overflow sidebar — scroll into view before interact:**
```typescript
const zoneBtn = page.locator('aside button').filter({ hasText: 'USDA Zone' })
await zoneBtn.scrollIntoViewIfNeeded()
await zoneBtn.click()
```

**Nav link scoping (avoid footer ambiguity):**
```typescript
// Footer always renders Sign Up, Sign In, My Lists regardless of auth
// Scope to nav to avoid strict mode violations
await expect(page.locator('nav').getByRole('link', { name: 'Sign up' })).toBeVisible()
// OR scope to paragraph for form-level links
await page.locator('p').getByRole('link', { name: 'Sign up' }).click()
```

**Active tab assertion:**
```typescript
const plantsTab = page.getByText('Plants', { exact: true }).first()
await expect(plantsTab).toHaveClass(/text-forest/)
```

**Plant card locator:**
```typescript
const card = page.locator('div.bg-cream.rounded-2xl').filter({ hasText: title })
```

**New tab assertions:**
```typescript
const [newPage] = await Promise.all([
  context.waitForEvent('page'),
  page.getByRole('link', { name: 'View presentation ↗' }).click(),
])
await expect(newPage).toHaveURL(/\/presents\//)
await newPage.close()
```

**Dialog (confirm) handling:**
```typescript
page.once('dialog', dialog => dialog.dismiss())
await page.locator('button[aria-label="Remove"]').first().click()
```

## Conditional Test Skipping

Tests that depend on optional database records use `test.skip`:
```typescript
test('shows invasive badge for invasive plant', async ({ page }) => {
  const { invasivePlantId } = getPlantIds()
  test.skip(!invasivePlantId, 'No invasive plant found — run import-permaculture to completion first')
  await page.goto(`/plants/${invasivePlantId!}`)
  // ...
})
```

## What Is Tested

| Area | Coverage |
|------|----------|
| Auth flows | Login (success + error), signup form, signout, auth guard redirect |
| Navigation | Logged-out nav links, link between pages |
| Plant browser | Search, filter by sun/water/type/zone/native state, combine filters, clear filters, empty state, count badge |
| Plant detail | Navigation to detail, back link, common/latin name display, invasive badge, notable cultivars section |
| Add to list (logged in) | Button visibility, dropdown with list names, "✓ Added" confirmation |
| My Lists | Create list (title only, title+desc), empty title guard, plant count badge, navigate to editor |
| List editor | Back link, plant items display, copy share link, view presentation link, remove plant (cancel) |
| Presentation page | Public access, plant count, plant cards, active tab, reports tab link, 404 for unknown shareId |
| Reports page | Public access, active tab, water/bloom/season tables, row ordering |

## What Is NOT Tested

- Any unit or component logic in isolation
- Server Actions or API routes directly (only tested via UI)
- Supabase RLS policies
- Password reset / email OTP flows (no test infrastructure for email)
- Drag-and-drop sort order on list editor
- Mobile filter drawer (tests run in desktop viewport)
- Image loading success/failure
- Permaculture-specific filter combinations (bloom month, dormancy, growth rate, layers, perm uses)
- Pagination on the plant browser

## Configuration Details

```typescript
// playwright.config.ts
export default defineConfig({
  testDir: './tests',
  timeout: 30000,
  retries: 1,              // 1 retry on failure (flake tolerance)
  reporter: 'list',
  use: {
    baseURL: 'https://permacultureplantpicker.com',
    screenshot: 'only-on-failure',
    video: 'retain-on-failure',
  },
  globalSetup: './tests/global-setup.ts',
  globalTeardown: './tests/global-teardown.ts',
})
```

**Timeouts:** Most `waitForSelector` calls use `{ timeout: 20000 }` for the initial plant load (Supabase cold start). Subsequent assertions use `{ timeout: 5000 }` or `{ timeout: 10000 }`. Default test timeout is 30 seconds.

## Key Gotchas

1. **Tests hit production.** Destructive operations (remove plant, delete list) must be scoped to `[TEST]` records. Never write a test that modifies real user data or the plant catalog.

2. **Filter sections start collapsed.** Always click the `aside button` header before asserting or interacting with checkboxes inside a `FilterSection`.

3. **Sidebar overflow requires scroll.** USDA Zone and Native State filter sections are below the scroll boundary — call `scrollIntoViewIfNeeded()` before clicking.

4. **Footer link ambiguity.** The footer always renders Sign Up, Sign In, and My Lists links unconditionally. Scope link assertions to `nav` or `p` to avoid Playwright strict mode errors.

5. **Auth state file must exist.** `tests/.auth-state.json` is created by `global-setup.ts`. If it doesn't exist (first run), the `logged-in` project will fail. Always run the full suite, not individual specs, on a fresh environment.

6. **`.env.test.local` required.** Must contain `TEST_USER_EMAIL` and `TEST_USER_PASSWORD` for global setup to authenticate.
