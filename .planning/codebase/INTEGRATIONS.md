# INTEGRATIONS
_Generated by gsd-codebase-mapper · focus: tech_

## Summary

This app integrates Supabase for database, auth, and email delivery; the Anthropic Claude API (claude-haiku) for AI-powered plant data enrichment; and the iNaturalist public API for taxonomy, common names, and CC-licensed plant photos. All third-party integrations are server-side only except Supabase, which is used on both client and server. The Flora API is a deferred integration (not yet active in production).

---

## Supabase (Database + Auth + Storage)

**Role:** Primary backend — PostgreSQL database, row-level security, user authentication, and transactional email delivery.

**SDK:**
- `@supabase/supabase-js` ^2.105.4 — data queries, auth operations
- `@supabase/ssr` ^0.10.3 — Next.js SSR cookie-based session management

**Client instantiation:**
- Server (RSCs, route handlers): `lib/supabase/server.ts` — `createServerClient` from `@supabase/ssr`, reads/writes cookies via `next/headers`
- Browser (client components): `lib/supabase/client.ts` — `createBrowserClient` from `@supabase/ssr`
- Scripts (admin/enrichment): direct `createClient` from `@supabase/supabase-js` with service role key

**Auth flow:**
- Email + password auth only (no OAuth providers enabled)
- Email confirmation disabled (`enable_confirmations = false` in `supabase/config.toml`)
- Password reset uses `token_hash` + `verifyOtp` — NOT PKCE code exchange (PKCE breaks when email opened in a different browser or when Chrome Safe Browsing prefetches the link)
- Auth callback route: `app/api/auth/callback/route.ts` — handles both `token_hash` (preferred) and `code` (PKCE fallback)
- JWT expiry: 1 hour; refresh token rotation enabled

**Email templates:**
- Branded HTML templates in `supabase/templates/`
- `confirm.html` — signup confirmation
- `reset-password.html` — password reset (uses `{{ .TokenHash }}` not `{{ .ConfirmationURL }}`)
- Templates must be kept in sync with `supabase/config.toml` `[auth.email.template.*]` sections AND the Supabase dashboard

**Email delivery:**
- Production: Resend SMTP via Namecheap DNS (configured in Supabase dashboard, not in repo config)
- Local dev: Supabase Inbucket (email testing server on port 54324, emails not actually sent)

**Database:**
- PostgreSQL 17 (local dev via Supabase CLI; production on Supabase hosted)
- Migrations in `supabase/migrations/` — applied via `supabase db push` (local) or Supabase dashboard (production)
- RLS enabled on all tables: `plants` (public read), `plant_lists` (owner-only), `plant_list_items` (owner-only)

**Environment variables:**
- `NEXT_PUBLIC_SUPABASE_URL` — required in app and all scripts
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — required in app (public, safe to expose)
- `SUPABASE_SERVICE_ROLE_KEY` — scripts only, never exposed client-side; also accepted as `SUPABASE_SECRET_KEY` (legacy alias)

## Anthropic Claude API

**Role:** AI enrichment of plant horticultural data during data import/backfill pipelines. Not used at runtime by the Next.js app.

**SDK:** `@anthropic-ai/sdk` ^0.96.0 (devDependency — scripts only)

**Model:** `claude-haiku-4-5-20251001`

**Usage patterns:**
- `scripts/import-plants.ts` — generates structured horticultural JSON (sun, water, plant_type, growth_rate, bloom_months, USDA zones, etc.) for each new plant
- `scripts/import-permaculture-plants.ts` — same enrichment for permaculture-specific batch imports
- `scripts/update-existing-plants.ts` — re-enriches existing plant records
- `scripts/retry-skipped-plants.ts` — retries plants that hit rate limits
- `scripts/backfill-usda-zones.ts` — parses `usda_zones` text to half-zone integers
- `scripts/backfill-native-states.ts` — infers US state codes from plant name + native range text

**Rate limiting:** 10 concurrent calls per batch with 15-second delay between batches (≈40 req/min, under the 50 req/min Haiku limit). Never remove the delay.

**Environment variable:**
- `ANTHROPIC_API_KEY` — scripts only, sourced from `.env.local`

## iNaturalist API

**Role:** Source of plant taxonomy (latin name, common name) and CC-licensed plant photos. No API key required — public API, unauthenticated.

**Base URL:** `https://api.inaturalist.org/v1/taxa`

**Usage:** Scripts only — not called at runtime by the Next.js app.

**Scripts that call iNaturalist:**
- `scripts/import-plants.ts` — fetches paginated species list sorted by observation count; 200 results/page, up to 6 pages (≈1,200 candidates)
- `scripts/import-permaculture-plants.ts` — fetches photo URL by species name
- `scripts/retry-skipped-plants.ts` — re-fetches pages to find gaps from prior rate-limit failures
- `scripts/fix-plant-images.ts` — replaces broken Wikimedia URLs with iNaturalist CDN photos

**Rate limiting:** 1 request/second (1,100ms delay enforced per iNaturalist API terms). User-Agent header identifies the project.

**Image CDN hosts (also used at runtime in the Next.js app):**
- `inaturalist-open-data.s3.amazonaws.com` — primary CDN
- `static.inaturalist.org` — alternate CDN
- Both are allowlisted in `next.config.mjs` `images.remotePatterns`

**No API key required.** Public read-only access.

**Role:** County-level plant native distribution data. Not active in production.

**Environment variable:**
- `FLORA_API_KEY` — scripts only, sourced from `.env.local`; not currently set in production
