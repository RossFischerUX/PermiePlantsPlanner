# Codebase Structure
_Generated by gsd-codebase-mapper · focus: arch_

## Summary

Next.js 14 App Router project with a single `app/` tree, shared utilities in `lib/`, data enrichment scripts in `scripts/`, and Supabase schema in `supabase/migrations/`. Routes are organized into two groups: `(app)` for the full nav+footer shell and `presents/` for standalone public pages.

## Key File Locations

**Entry Points:**
- `app/layout.tsx` — root HTML shell; sets font CSS variables and `bg-parchment`
- `app/(app)/layout.tsx` — all app pages; sticky nav with `auth.getUser()` server fetch
- `app/(app)/page.tsx` — marketing home (RSC)

**Routing:**
- `app/(app)/plants/page.tsx` — `/plants` browser
- `app/(app)/plants/[id]/page.tsx` — `/plants/[id]` detail
- `app/(app)/lists/page.tsx` — `/lists` dashboard
- `app/(app)/lists/[id]/page.tsx` — `/lists/[id]` editor
- `app/presents/[shareId]/page.tsx` — `/presents/[shareId]` public grid
- `app/presents/[shareId]/reports/page.tsx` — `/presents/[shareId]/reports` tables
- `app/api/auth/callback/route.ts` — `/api/auth/callback` token exchange

**Shared Types and Utilities:**
- `lib/types.ts` — all TypeScript interfaces (`Plant`, `PlantList`, `PlantListItem`)
- `lib/zones.ts` — USDA zone encode/decode helpers; import `ZONE_LABELS` for filter options
- `lib/us-states.ts` — `US_STATES` array for the native state filter dropdown
- `lib/supabase/server.ts` — use in RSCs and route handlers
- `lib/supabase/client.ts` — use in `'use client'` components

**Configuration:**
- `tailwind.config.ts` — Botanical Heritage color tokens (`forest`, `parchment`, `cream`, etc.) and `shadow-warm`/`shadow-warm-md`
- `next.config.mjs` — allowed remote image hostnames (must update when adding new image sources)
- `tsconfig.json` — `@/*` path alias maps to project root; `scripts/` excluded from build

**Database:**
- `supabase/migrations/` — all schema changes as ordered SQL files
- `supabase/templates/` — branded email HTML (confirmation, recovery)

## Naming Conventions

**Files:**
- Route pages: `page.tsx` (Next.js convention)
- Layout files: `layout.tsx` (Next.js convention)
- Client islands co-located with their page: PascalCase descriptive name, e.g. `AddToListClient.tsx`, `ListItemActions.tsx`, `NewListForm.tsx`, `CopyShareUrl.tsx`
- Lib utilities: kebab-case, e.g. `us-states.ts`, `zones.ts`

**Directories:**
- Route groups: lowercase with parens, e.g. `(app)/`
- Dynamic segments: `[id]`, `[shareId]`
- Feature groupings: plural noun, e.g. `plants/`, `lists/`, `auth/`

## Where to Add New Code

**New page inside the nav/footer shell:**
- Add directory under `app/(app)/`
- Create `page.tsx` — use RSC by default; only add `'use client'` if interactive
- If the page needs auth, add `redirect('/auth/login')` guard at the top (see `app/(app)/lists/page.tsx`)

**New public standalone page (no nav/footer):**
- Add directory directly under `app/`, outside `(app)/` — e.g. `app/embed/[id]/page.tsx`

**New interactive widget co-located with a page:**
- Create `ComponentName.tsx` in the same directory as `page.tsx`
- Add `'use client'` at top, import `lib/supabase/client.ts`
- After mutations call `router.refresh()` to re-run the parent RSC

**New shared utility / data helper:**
- Add to `lib/` as a kebab-case `.ts` file
- Import via `@/lib/filename`

**New plant field:**
1. Write a migration in `supabase/migrations/` with the next timestamp prefix
2. Add the field to `lib/types.ts` `Plant` interface
3. If filterable: add filter option constant in `app/(app)/plants/page.tsx` and filter logic in `applyFilters`
4. Display in `app/(app)/plants/[id]/page.tsx` (and optionally `app/presents/[shareId]/page.tsx`)
5. If backfill needed: add a script to `scripts/`

**New database table:**
1. Add migration to `supabase/migrations/` (include RLS policies)
2. Add TypeScript interface to `lib/types.ts`

**New E2E test:**
- Add `.spec.ts` to `tests/`
- Assign to the `logged-in` or `logged-out` project in `playwright.config.ts`

## Module Boundaries

| Module | Can import from | Cannot import from |
|--------|-----------------|-------------------|
| `app/(app)/**` RSC | `lib/`, `@/lib/supabase/server` | `@/lib/supabase/client` |
| `app/(app)/**` `'use client'` | `lib/`, `@/lib/supabase/client` | `@/lib/supabase/server` |
| `app/presents/**` | `lib/`, `@/lib/supabase/server` | `@/lib/supabase/client` |
| `app/api/**` | `lib/`, `@/lib/supabase/server` | `@/lib/supabase/client` |
| `lib/` | other `lib/` files | `app/` (no circular deps) |
| `scripts/` | `lib/`, external packages | `app/` (excluded from TS build) |
